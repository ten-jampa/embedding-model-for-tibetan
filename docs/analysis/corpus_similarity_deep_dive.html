Why this matters

The actual question is historical, not merely computational

The motivating problem is whether two Tibetan corpora from different sectarian settings contain evidence of textual borrowing, reuse, or unusually strong semantic overlap. That sounds like a similarity problem, but it is really a layered inference problem. A high score between two sentences may reflect genuine reuse, shared doctrinal vocabulary, shared liturgical opening style, or a broad semantic field that many texts in the tradition inhabit.

This is worth knowing: when the historical question is stronger than the statistical instrument, the honest job of the pipeline is to reduce the search space and surface suspicious regions. It should not pretend to replace philology.

Working thesis: the system should optimize for reviewability, robustness, and traceability, not just raw similarity ranking.

System map

What the current pipeline actually does

1. Segment Tibetan text

Each text is segmented into Tibetan sentence-like units using the current botok_ours engine. This is not a trivial preprocessing step. In this project, the segmentation engine is the first model in the chain, because bad segment boundaries distort everything downstream.

2. Embed each sentence

Every segment is sent through Gemma-Mitra and represented as a normalized vector using the last non-padding token hidden state. Those vectors are the atomic objects out of which all later similarity scores are built.

3. Compare all sentences across texts

For a text pair with N and M segments, the system produces an N × M similarity matrix. At corpus scale, this becomes a grid of text-pair matrices across the two directories.

Text A → [a₁, a₂, ... aₙ] and Text B → [b₁, b₂, ... bₘ] ⇒ S[i, j] = cosine(embed(aᵢ), embed(bⱼ))

The inner loop already exists in the SDK. The missing engineering layer is the outer orchestration: scan folders, deduplicate files, persist embeddings, generate pairwise matrices, and summarize them into a macro-level map.

Key design decision

Why Gemma-Mitra introduced a methodological fork

Model-card retrieval mode

Gemma-Mitra was published as an asymmetric retrieval model. In that setup, the query side is wrapped with an instruction prefix and the corpus side is left as raw text. The current SDK default follows that design.

Neutral peer-comparison mode

The project question sounds more like sentence-to-sentence comparison between peers than a search query against a candidate corpus. That is why we tested query_corpus, query_query, and raw_raw instead of inheriting retrieval mode blindly.

Important distinction: cosine similarity itself is symmetric, but the embedding protocol may not be. The asymmetry enters before cosine is computed.

What we learned empirically

Current evidence from the corpus audit and notebook work

Segmentation looks usable, not solved

The corpus-wide audit suggested mostly reasonable segment lengths. One corpus produced 4,967 segments with median length 43 characters and only a handful of long outliers. The second corpus produced 3,837 segments with median length 39 characters and almost no extreme segments. That supports a first-pass run, but not blind confidence.

Exact duplicate handling is necessary

The input directories already contain exact duplicate files and an older-versions subfolder. That means the outer runner must detect duplicates by hash, retain one canonical witness, record exclusions, and keep a provenance report.

Protocol differences changed numbers more than rankings

On both the synthetic paragraph test and the later real-passage test, the strongest matches were broadly stable across query_corpus, query_query, and raw_raw. Score magnitudes moved, but the alignment pattern did not collapse.

8-bit Colab inference is viable for exploration

A T4 runtime could load the 9B model in 8-bit with about 9.5 GB allocated GPU memory after warmup. The first load was slow, but subsequent single-sentence embeddings were sub-second. That is enough for protocol analysis and limited batch runs, though AWS is still cleaner for a full reproducible corpus job.

Most important qualitative result: the top sentence matches were often meaningful, but many looked meaningful at the level of shared doctrinal-opening rhetoric rather than clear textual reuse. That is exactly the kind of ambiguity the final system must preserve rather than hide.

Notebook signal

What the first real-passage comparison suggested

Four real passage pairs were compared across the sectarian corpora. The strongest pair was the one that also looked strongest to a human before running the model: two elevated opening passages with homage, exalted doctrinal framing, and proclamation-like rhetoric. The weakest pair was the one with the clearest genre mismatch: practical contemplative instruction versus grand doctrinal opening.

Text pair Observed pattern Interpretation
A1 vs B2 Highest max scores across most protocols Likely driven by shared rhetorical mode and doctrinal opening structure
A1 vs B1 Mid-strength similarity Shared conceptual field, less direct rhetorical alignment
A2 vs B1 Mid-strength similarity Potentially more interesting philosophically than liturgically
A2 vs B2 Weakest overall Genre mismatch matters; not all high-register texts align equally
Lesson: the model is not returning random noise, but it is also not automatically isolating reuse. It is partly sensitive to genre, rhetoric, and doctrinal register.

Macro statistics

How microscopic scores should color the macroscopic space

The atomic object is an N × M sentence similarity matrix for one text pair. The right statistical move is not to squash that to a single number immediately. Instead, summarize each matrix with a small family of features that capture different structural signals.

Summary What it captures Main weakness
max(S) A single striking parallel Too fragile; one accidental hit can dominate
mean(S) Overall semantic proximity Diluted when most sentence pairs are unrelated
meanᵢ maxⱼ S[i,j] How well each sentence in A finds a best match in B Can still reward generic shared language
meanⱼ maxᵢ S[i,j] The reverse directional coverage Needs interpretation alongside the row-wise version
p95(S) or p99(S) Upper-tail strength without relying on one maximum Threshold behavior can still be genre-sensitive
top-k mean A handful of strongest local parallels K must be chosen deliberately

After that first layer, the next important signals are structural rather than purely numeric: density above threshold, clustered high-similarity regions, and whether matched sentences preserve any rough ordering. A genuine borrowing-like relationship tends to produce more local coherence than a handful of scattered peaks.

Recommended first-pass text-pair profile = [max, p95, mean_best_row, mean_best_col, top_k_mean, high_score_density, block_coherence]

Statistical discipline

The missing ingredient is a background distribution

A score like 0.68 does not mean much in the abstract. It only means something relative to this model, this segmentation pipeline, and this genre. That is why the strongest corpus-level next step is not just ranking text pairs; it is comparing cross-corpus score distributions against baselines.

Cross-corpus baseline

Compare the distribution of cross-sect text-pair summaries against one another.

Within-corpus baseline

Compare cross-corpus text pairs against within-corpus pairs from each side.

Permutation-style control

Optionally compare against shuffled or reassigned sentence structures to estimate chance.

Plain-language version: the real question is not “is this score high?” It is “is this score unusually high for this kind of Tibetan material?”

Paper lens

What the Keutzer paper contributes to this project

The paper on Mind-section literature is useful less as a code recipe than as a methodological model. It treats text analytics as exploratory distant reading that narrows the field for later close reading. That is the right posture for this project too.

Where the paper aligns with this work

It asks sectarian and genre questions using corpus comparison, similarity analysis, and clustering. It also highlights that different Tibetan corpora can share topical commonality while still being distinguishable in style and vocabulary.

Where the paper warns us to stay cautious

In its discussion of possible borrowing, cosine similarity helped surface a candidate but could not settle directionality or layered textual history. That warning transfers directly to our embedding setup.

Best takeaway from the paper: quantitative similarity is good for finding where to read hard, not for skipping the hard reading.

Pitfalls

What can go wrong if the system is used naively

Shared sacred register can masquerade as reuse

A model can be impressed by invocations, stock epithets, and doctrinal opening language even when the real textual relationship is weak.

Bad segmentation quietly corrupts downstream evidence

If the sentence unit is wrong, the embedding unit is wrong, and the similarity matrix becomes harder to interpret no matter how good the model is.

Retrieval asymmetry may bias interpretation

If one side is encoded as a query and the other as corpus text, differences in role encoding can affect scores even when the underlying sentence meanings are similar.

One-number corpus summaries are misleading

A single maximum score or average collapses away whether the evidence is isolated, sustained, directional, or structurally coherent.

Execution plan

Recommended next steps

  1. Build the corpus orchestration layer around the SDK: folder scan, exact-dedup by hash, manifests, and exclusion reports.
  2. Persist one embedding artifact per retained file, so the expensive step is done once and the full run is resumable on AWS.
  3. Compute one sentence matrix per text pair and store both the full matrix and a ranked top-k evidence table.
  4. Summarize each matrix with a compact profile: max, p95, mean_best_row, mean_best_col, top_k_mean, and a simple coherence feature.
  5. Run both a canonical protocol and at least one robustness protocol on a sample, so the final method is not overfit to one formatting choice.
  6. Compare cross-corpus distributions against within-corpus baselines before making any historical claims.
  7. Manually inspect the highest-value matches, especially the ones that look content-bearing rather than merely liturgical or generic.
Recommended output of the finished system: a ranked corpus map of suspicious text pairs, each linked to the exact sentences and passages that caused the ranking.

Open questions

What still needs to be verified

  • Whether symmetric encoding changes any top-ranked file pairs once the sample grows beyond a few passages.
  • How often the 512-token limit actually truncates real Tibetan segments in the full corpus run.
  • Whether a simple block-coherence statistic is enough, or whether a more explicit passage-alignment model is needed.
  • How much 8-bit inference changes rankings relative to a higher-precision run on stronger hardware.
  • Which top-k matches remain interesting after filtering out formulaic doctrinal-opening language.
